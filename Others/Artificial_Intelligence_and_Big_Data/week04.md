[머신러닝 개요]
<머신러닝의 역사>
1959 아서 새무얼이 최초로 정의 '프로그램을 명시적으로 작성하지 않고, 컴퓨터에 학습 능력을 부여'
1998 톰 미첼이 구체적으로 정의 '프로그램은 경험으로부터 학습한다.'

AI: 스스로 생각하고 판단, 알려준 것 이상을 처리
ML: 컴퓨터가 데이터로부터 학습, 경험으로부터 지식을 학습
_________________________________________________________________________________________________________________________
[지도학습]
지도학습: 주어진 입력과 출력 사이의 대응 관계를 학습 (과적합, 부적합 주의!!!)
1) 분류(Classification): 특성에 따라 입력을 **구분**짓는 모델
	- 이항분류: 2가지 클래스로 분류
	- 다항분류: 3가지 이상의 클래스로 분류
2) 회귀(Regression): 입출력 사이의 **관계**를 대표하는 모델
_________________________________________________________________________________________________________________________
[대표적인 분류 방법]
K-NN (K-Nearest Neighbor) 분류 알고리즘
- 1950년대 개발
- 가장 가까운 것들과의 거리 계산으로 클래스 분류
- 새로운 입력 데이터와 가장 가까운 k개의 이웃 데이터 선택
	장점: 매우 간단, 빠르고 효과적, 어떤 데이터라도 유사성 측정 가능
	단점: 적절한 k값을 선택해야 함
* 노드 분할을 잘 하는 것이 핵심!!

SVM (Support Vector Machine) 분류 알고리즘
- 1990년 개발
- 통계 학습 이론의 결과 기반
- 새로운 데이터가 어느 영역에 속하는지 판단
- 가장 큰 폭을 가진 하나의 경계선을 찾는 알고리즘
- used in 패턴인식, 자료분석
	1) Ouliner가 섞여 있는 경우
		- Hard margin
		- Soft margin
	2) 3개 이상의 속성 => 차원을 늘려 평면으로 구분
	3) 직선 분류 어려운 경우 => 곡선 분류도 가능

결정트리 (Desicion Tree)
- 판단 분류를 위한 지식을 트리형태로 나타낸 것
- 관측값과 목표값을 연결하는 예측 모델
- like '스무고개'
* 노드 분할을 잘 하는 것이 핵심!!

앙상블 분류기 (Ensemble Classifier)
- 여러 클래스를 만든 후, 투표 or 가중치 투표로 결합
	1) 배깅: 여러 클래스를 만든 후,  투표 or 가중치 투표로 결합하는 앙상블 분류기 생성 방법
		1) 랜텀 포레스트: 결정트리를 활용한 배깅 방법
	2) 부스팅: k개의 클래스를 순차적으로 만드는 앙상블 분류기 생성 방법

<데이터>
정형데이터: 행과 열로 나타내어짐 => 앙상블 방법의 성능이 좋음
비정형데이터: 사진, 동영상, 음성신호, 센서신호, 텍스트 등
